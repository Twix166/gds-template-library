# Purpose

Define the minimum viable service, why it is viable and what it intentionally excludes.

# When to use this

Use during Alpha when shaping Beta scope and release strategy.

# Inputs

- User needs
- Research findings
- Technical options
- Policy and operational constraints

# Template

Copy this structure when shaping a Beta proposal. The MVP should still solve a coherent problem for users.

# Quality checklist

- The MVP still solves a real user problem.
- Exclusions and risks are explicit.
- Measures show how viability will be assessed.

# Field glossary

Hover over table headers, and some field-name cells, to see short definitions in renderers that support HTML title text.

- **Acceptance signal**: The observable result that shows the capability is acceptable.
- **Baseline or evidence**: The current baseline or evidence supporting the entry.
- **Beta decision supported**: The Beta planning or approval decision this MVP definition supports.
- **Check**: A viability check used to assess whether the MVP can proceed.
- **Core outcome**: The main result the MVP must achieve for users or the service.
- **Evidence**: The research, data, observation or source that supports the statement.
- **Excluded capability**: A capability deliberately excluded from the MVP.
- **Field**: The name of the information item to complete.
- **Future review point**: When or how the excluded item should be reconsidered.
- **Included capability**: A feature, journey, support route or capability included in the MVP.
- **MVP name**: The name or short label for the minimum viable service.
- **Measure or risk**: A success measure or risk linked to the MVP.
- **Owner**: The person or role accountable for the item.
- **Primary user groups**: The main user groups the MVP is intended to serve.
- **Reason excluded**: Why the capability is excluded from the MVP.
- **Response**: The content to complete for the field.
- **Risk**: An uncertain event or condition that could affect outcomes.
- **Status**: The current state of the item.
- **Target or mitigation**: The target result or mitigation action.
- **User need served**: The evidenced user need served by the capability.
- **User problem solved**: The user problem the MVP is intended to solve.
