# Purpose

Define the minimum viable service, why it is viable and what it intentionally excludes.

# When to use this

Use during Alpha when shaping Beta scope and release strategy.

# Inputs

- User needs
- Research findings
- Technical options
- Policy and operational constraints

# Template

Add outcomes, included scope, excluded scope, assumptions, measures, risks and release implications.

# Quality checklist

- The MVP still solves a real user problem.
- Exclusions and risks are explicit.
- Measures show how viability will be assessed.
