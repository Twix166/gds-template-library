# Purpose

Summarise findings from user testing and the implications for service design.

# When to use this

Use after each prototype test, usability test or research round.

# Inputs

- Research notes
- Session recordings or transcripts
- Prototype description
- Observation notes

# Template

Add research goals, participants, findings, severity, evidence, recommendations and decisions.

# Quality checklist

- Findings distinguish observation from interpretation.
- Recommendations are prioritised.
- Design or backlog changes are traceable.
