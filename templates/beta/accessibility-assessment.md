# Purpose

Assess accessibility evidence, risks, issues and remediation actions.

# When to use this

Use during Beta before significant releases and assessment preparation.

# Inputs

- Accessibility audit findings
- User research with disabled users
- Design and content evidence
- Defect register

# Template

Add scope, standards considered, test methods, findings, severity, remediation, owners and evidence links.

# Quality checklist

- Evidence includes automated, manual and user-based testing where appropriate.
- Issues have owners and dates.
- Known limitations and mitigations are transparent.
