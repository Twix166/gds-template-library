# Purpose

Define support channels, roles, service levels, escalation routes and responsibilities.

# When to use this

Use before users rely on the service and update as support arrangements change.

# Inputs

- Service blueprint
- Operational readiness checklist
- User support insight
- Organisational support policies

# Template

Add support scope, channels, hours, roles, escalation, service levels, reporting and improvement loop.

# Quality checklist

- Users know how to get help.
- Support responsibilities and escalation routes are clear.
- Support data feeds service improvement.
