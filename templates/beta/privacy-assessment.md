# Purpose

Assess personal data use, privacy risks, lawful basis, mitigations and evidence.

# When to use this

Use when the service processes personal data or changes how personal data is handled.

# Inputs

- Data dictionary
- Service blueprint
- Security risk assessment
- Information governance advice

# Template

Add data flows, personal data categories, lawful basis, risks, mitigations, retention and approvals.

# Quality checklist

- Personal data use is necessary and proportionate.
- Risks and mitigations are owned.
- Links to formal DPIA or privacy notices are included where relevant.
