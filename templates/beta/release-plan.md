# Purpose

Plan how service changes will be released, communicated, monitored and rolled back if needed.

# When to use this

Use before private beta, public beta or significant live releases.

# Inputs

- Beta delivery plan
- Test strategy
- Operational readiness checklist
- Communications plan

# Template

Use this structure for private beta, public beta, Live transition or significant releases.

# Quality checklist

- Release risks and rollback approach are clear.
- Operational and support teams are ready.
- Success measures and monitoring are defined.

# Field glossary

Hover over table headers, and some field-name cells, to see short definitions in renderers that support HTML title text.

- **Activity**: A release, delivery or assurance activity that needs to happen.
- **Communication route**: How users, staff or stakeholders will be informed.
- **Evidence or output**: The output produced or evidence recorded by the activity.
- **Field**: The name of the information item to complete.
- **Go/no-go decision owner**: The person accountable for deciding whether the release proceeds.
- **Owner**: The person or role accountable for the item.
- **Release date**: The date the release is planned or completed.
- **Release item**: A feature, change or operational activity included in the release.
- **Release name**: The name or identifier for the release.
- **Release owner**: The person accountable for the release.
- **Release type**: The type of release, such as private beta, public beta or live.
- **Response**: The content to complete for the field.
- **Risk**: An uncertain event or condition that could affect outcomes.
- **Rollback or mitigation**: The rollback step or mitigation used if a release scenario occurs.
- **Scenario**: A realistic situation used for testing or planning.
- **Status**: The current state of the item.
- **Test evidence**: Evidence generated through testing.
- **Timing**: When the activity should happen.
- **Trigger**: The event or condition that starts a procedure.
- **User value**: The expected value for users.
