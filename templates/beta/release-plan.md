# Purpose

Plan how service changes will be released, communicated, monitored and rolled back if needed.

# When to use this

Use before private beta, public beta or significant live releases.

# Inputs

- Beta delivery plan
- Test strategy
- Operational readiness checklist
- Communications plan

# Template

Add release scope, dates, environments, approvals, communications, monitoring, rollback and success criteria.

# Quality checklist

- Release risks and rollback approach are clear.
- Operational and support teams are ready.
- Success measures and monitoring are defined.
