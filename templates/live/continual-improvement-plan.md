# Purpose

Plan ongoing service improvements based on evidence, priorities and capacity.

# When to use this

Use throughout Live to manage improvements beyond routine support.

# Inputs

- Service dashboard
- User feedback
- Service reviews
- Backlog and roadmap

# Field glossary

Hover over table headers, and some field-name cells, to see short definitions in renderers that support HTML title text.

- **Decision needed**: The decision required to move the item forward.
- **Dependency**: Another team, decision, system or event the item depends on.
- **Due date**: The date by which an action or condition should be completed.
- **Effort**: The estimated size, complexity or effort needed.
- **Evidence**: The research, data, observation or source that supports the statement.
- **ID**: A stable identifier used to track the item across documents.
- **Improvement ID**: A unique identifier for the improvement item.
- **Improvement opportunity**: A potential change that could improve user outcomes or service performance.
- **Measure of success**: How the team will know whether an improvement worked.
- **Next action**: The immediate action needed to progress the item.
- **Owner**: The person or role accountable for the item.
- **Priority**: The relative importance of the item.
- **Reason for priority**: Why the improvement has its current priority.
- **Review date**: The date an item should be reviewed.
- **Status**: The current state of the item.
- **User or service outcome**: The outcome for users or the service.
