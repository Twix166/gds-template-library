# Purpose

Review live service performance, risks, issues, user feedback and improvement priorities.

# When to use this

Use at regular service governance meetings or major review points.

# Inputs

- Service dashboard
- Incident reports
- User research or feedback
- Risk and issue registers

# Template

Add performance summary, user insight, operational issues, risks, decisions, actions and next review date.

# Quality checklist

- Review uses current evidence.
- Decisions and actions have owners.
- Improvement priorities are linked to user and service outcomes.

# Related templates

- Service dashboard
- Risk register
- Continual improvement plan
