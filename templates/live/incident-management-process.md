# Purpose

Define how incidents are detected, assessed, managed, communicated and reviewed.

# When to use this

Use before launch and keep updated throughout Live operations.

# Inputs

- Support model
- Operational runbook
- Monitoring arrangements
- Organisational incident policy

# Template

Add incident types, severity levels, roles, response steps, communications, escalation and post-incident review.

# Quality checklist

- Severity levels and response expectations are clear.
- Communication responsibilities are defined.
- Reviews lead to improvement actions.
