# Purpose

Define live service measures, data sources, reporting cadence and interpretation.

# When to use this

Use when establishing or reviewing service performance monitoring.

# Inputs

- Success measures
- Analytics
- Operational data
- Support data

# Template

Add measure, definition, data source, owner, frequency, target, trend and action threshold.

# Quality checklist

- Measures reflect user outcomes and service health.
- Data definitions are clear.
- Measures lead to decisions or improvement actions.
