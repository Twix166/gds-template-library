# Purpose

Track realised benefits against expected outcomes, measures and evidence.

# When to use this

Use after launch and at agreed benefit review points.

# Inputs

- Benefits register
- Service dashboard
- Business case
- Operational and financial data

# Quality checklist

- Benefits are measured against baselines where available.
- Variance is explained.
- Actions are linked to improvement planning.

# Field glossary

Hover over table headers, and some field-name cells, to see short definitions in renderers that support HTML title text.

- **Action**: The specific follow-up task needed to progress, validate or close an item.
- **Baseline**: The current or starting value used to compare future change.
- **Benefit**: An expected positive outcome from the service.
- **Benefit owner**: The person accountable for tracking and realising the benefit.
- **Business case or benefits register link**: A link to the source where expected benefits were originally defined.
- **Current result**: The most recent measured value.
- **Evidence**: The research, data, observation or source that supports the statement.
- **Explanation**: The reason for the result, variance or conclusion.
- **Field**: The name of the information item to complete.
- **Follow-up decision**: The decision needed after reviewing the benefit evidence.
- **Measure**: A metric or indicator used to judge performance or success.
- **Reporting period**: The time period covered by the report.
- **Response**: The content to complete for the field.
- **Service name**: The name of the service being documented.
- **Status**: The current state of the item.
- **Target**: The intended value or threshold for a measure.
- **Variance from target**: The difference between the target and the current result.
