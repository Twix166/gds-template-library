# Purpose

Compare potential options against user, policy, technical, operational and delivery criteria.

# When to use this

Use when deciding whether and how to proceed after Discovery.

# Inputs

- Problem statement
- User needs
- Architecture findings
- Constraints and risks

# Template

Add options, assessment criteria, evidence, benefits, risks, costs, dependencies and recommendation.

# Quality checklist

- Options include doing nothing or stopping where relevant.
- Criteria are explicit and balanced.
- The recommendation explains trade-offs.
