# Purpose

Map how the current service works across user actions, channels, processes, teams and systems.

# When to use this

Use when the team needs to understand the existing service before designing changes.

# Inputs

- Journey maps
- Process maps
- Operational interviews
- System information

# Template

Copy this structure. A current-state blueprint should show how the service works today across channels, teams, systems and policies.

# Quality checklist

- The blueprint shows both user-facing and internal activity.
- Manual workarounds and failure demand are captured.
- Evidence sources are linked.
