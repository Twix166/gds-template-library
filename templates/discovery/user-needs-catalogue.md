# Purpose

Record user groups, user needs, evidence sources and confidence levels.

# When to use this

Use throughout Discovery and update as research generates new evidence.

# Inputs

- Research notes
- Analytics and service data
- Support queries
- Existing user insight

# Template

Copy this structure and add one row for each evidenced need. Do not record stakeholder requests or feature ideas as user needs.

# Quality checklist

- Needs describe what users need to do, not features.
- Each need links to evidence.
- Confidence and gaps are visible.
