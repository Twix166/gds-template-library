# Purpose

Capture the current user journey, pain points, needs and opportunities for improvement.

# When to use this

Use during Discovery to build a shared understanding of current user experience.

# Inputs

- User research findings
- Analytics
- Complaints or support data
- Existing process documentation

# Template

Copy this structure. Keep the journey from the user's point of view and show where evidence comes from.

# Quality checklist

- Journey stages reflect real user behaviour.
- Pain points are supported by evidence.
- Opportunities do not jump straight to a single solution.
