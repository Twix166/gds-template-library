# Purpose

Link user needs, requirements, design decisions, tests and evidence.

# When to use this

Use when the team needs to show how evidence connects across delivery artefacts.

# Inputs

- User needs catalogue
- Requirements catalogue
- Test strategy
- Decision log

# Template

Add user need, requirement, design artefact, test evidence, decision, status and evidence links.

# Quality checklist

- Links are useful and maintained.
- Gaps between needs, requirements and evidence are visible.
- Traceability supports decisions rather than becoming a burden.
