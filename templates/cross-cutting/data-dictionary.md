# Purpose

Define data entities, fields, ownership, sensitivity, standards and usage.

# When to use this

Use when the service collects, stores, exchanges or reports data.

# Inputs

- Service blueprint
- Architecture overview
- Privacy assessment
- Data standards

# Template

Add entity, field, definition, format, owner, source, sensitivity, retention, standard and usage notes.

# Quality checklist

- Definitions are consistent and plain.
- Sensitive data is clearly marked.
- Standards and sources are recorded.
