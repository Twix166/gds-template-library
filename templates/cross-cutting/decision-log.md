# Purpose

Record key service, delivery, design, technical and governance decisions.

# When to use this

Use whenever a decision has meaningful impact or may need to be understood later.

# Inputs

- Governance meetings
- Architecture decision records
- Research findings
- Delivery reviews

# Template

Add decision ID, date, decision, context, options, rationale, owner, consequences and review date.

# Quality checklist

- Decisions include enough context to be understood later.
- Evidence and rationale are linked.
- Consequences and follow-up actions are visible.
