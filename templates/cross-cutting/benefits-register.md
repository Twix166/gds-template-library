# Purpose

Track expected benefits, measures, owners, baselines and evidence.

# When to use this

Use when the service has expected policy, operational, financial or user benefits.

# Inputs

- Business case
- Service objectives
- Success measures
- Operational data

# Template

Add benefit ID, description, type, owner, measure, baseline, target, evidence source, status and review date.

# Quality checklist

- Benefits are measurable where possible.
- Owners and evidence sources are clear.
- Benefits are reviewed after launch.

# Related templates

- Service dashboard
- Benefits realisation report
- Continual improvement plan
